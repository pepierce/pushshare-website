# GitHub Pages: Public vs Private 저장소

## 📊 옵션 비교

| 저장소 타입 | GitHub Pages 사용 | 비용 | 웹사이트 접근 |
|------------|------------------|------|--------------|
| **Public** | ✅ 무료 사용 가능 | 무료 | 모든 사용자 접근 가능 |
| **Private** | ⚠️ GitHub Pro 필요 | $4/월 | 모든 사용자 접근 가능 (웹사이트는 여전히 Public) |

## 🎯 핵심 포인트

### 중요한 사실
- **웹사이트 자체는 항상 Public입니다**
  - 저장소가 Private이어도, GitHub Pages로 배포된 웹사이트는 누구나 접근 가능
  - 저장소를 Private으로 해도 웹사이트 보안에는 도움이 되지 않음

### 저장소를 Public으로 해도 안전한 이유
1. ✅ **앱 코드는 별도 저장소에 있음**
   - 실제 앱 코드는 Bitbucket에 있음
   - GitHub에는 웹사이트 파일만 있음

2. ✅ **웹사이트 파일은 공개되어야 함**
   - 홈페이지, 개인정보처리방침, 이용약관 등은 공개되어야 정상
   - `app-ads.txt`도 Google이 접근해야 하므로 Public이어야 함

3. ✅ **민감한 정보가 없음**
   - 웹사이트 파일에는 API 키나 비밀번호 같은 정보가 없음
   - 단순 HTML, CSS 파일만 있음

## 💡 추천

### Public 저장소 사용 (권장) ⭐

**이유:**
- ✅ 완전 무료
- ✅ GitHub Pages 무료 사용
- ✅ 웹사이트 파일은 어차피 공개되어야 함
- ✅ 앱 코드는 별도 저장소(Bitbucket)에 있음

**저장소에 있는 것:**
- `index.html` - 홈페이지 (공개되어야 함)
- `styles.css` - 스타일시트 (공개되어야 함)
- `app-ads.txt` - Google AdMob 파일 (공개되어야 함)
- `privacy-policy.html` - 개인정보처리방침 (공개되어야 함)
- `terms-of-service.html` - 이용약관 (공개되어야 함)

**민감한 정보:**
- ❌ API 키 없음
- ❌ 비밀번호 없음
- ❌ 앱 소스 코드 없음 (별도 저장소에 있음)

## 🔒 Private 저장소를 원한다면

### 옵션 1: GitHub Pro 사용 ($4/월)
- Private 저장소에서도 GitHub Pages 사용 가능
- 하지만 웹사이트는 여전히 Public으로 접근 가능

### 옵션 2: 다른 호스팅 서비스 사용
- Netlify: Private 저장소에서도 무료 사용 가능
- Vercel: Private 저장소에서도 무료 사용 가능

## 🎯 결론

**웹사이트 저장소는 Public으로 두는 것이 정상입니다.**

- 홈페이지는 공개되어야 함
- 개인정보처리방침, 이용약관도 공개되어야 함
- `app-ads.txt`도 Google이 접근해야 함
- 앱 코드는 별도 저장소(Bitbucket)에 있으므로 안전

저장소를 Public으로 두셔도 전혀 문제없습니다! 🎉
