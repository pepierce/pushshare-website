# GitHub 저장소 권한 및 보안

## 🔒 Public 저장소의 실제 권한

### ✅ 소유자(당신)만 할 수 있는 것
- ✅ 파일 수정 및 삭제
- ✅ 커밋 및 푸시
- ✅ 브랜치 생성 및 삭제
- ✅ 설정 변경
- ✅ 저장소 삭제

### ❌ 다른 사용자는 할 수 없는 것
- ❌ **직접 파일 수정 불가**
- ❌ **직접 커밋 불가**
- ❌ **직접 푸시 불가**
- ❌ **설정 변경 불가**

### ⚠️ 다른 사용자가 할 수 있는 것 (하지만 승인이 필요)
- ✅ **Fork** - 저장소를 복사 (읽기 전용)
- ✅ **Pull Request** - 수정 제안 (당신이 승인해야만 반영)
- ✅ **이슈 생성** - 버그 리포트나 제안

---

## 🛡️ 보안 설정

### 기본 보안 (이미 적용됨)
1. ✅ **소유자만 직접 수정 가능**
2. ✅ **Pull Request는 승인 필요**
3. ✅ **브랜치 보호 규칙 설정 가능**

### 추가 보안 설정 (선택사항)

#### 1. 브랜치 보호 규칙 설정
```
Settings > Branches > Add rule
- Require pull request reviews before merging
- Require status checks to pass
- Include administrators
```

#### 2. Collaborator 관리
```
Settings > Collaborators
- 특정 사람에게만 접근 권한 부여
- 기본적으로는 소유자만 접근 가능
```

#### 3. Pull Request 승인 필수
- 다른 사람이 수정을 제안해도
- 당신이 승인하기 전까지는 반영되지 않음

## 📋 실제 시나리오

### 시나리오 1: 다른 사용자가 파일을 수정하려고 시도
```
❌ 불가능: 직접 수정 불가
✅ 가능: Fork 후 자신의 저장소에서 수정
✅ 가능: Pull Request로 수정 제안 (당신이 승인해야만 반영)
```

### 시나리오 2: 악의적인 사용자가 파일을 삭제하려고 시도
```
❌ 불가능: 직접 삭제 불가
✅ 가능: Pull Request로 삭제 제안 (당신이 승인해야만 반영)
```

### 시나리오 3: 누군가 저장소를 복사
```
✅ 가능: Fork (읽기 전용 복사본)
⚠️ 영향: 원본 저장소에는 영향 없음
```

## 🔐 추가 보안 조치

### 1. 2단계 인증 활성화 (강력 권장)
```
GitHub Settings > Security > Two-factor authentication
- 계정 보안 강화
- 다른 사람이 당신의 계정으로 접근하는 것 방지
```

### 2. Personal Access Token 보안
```
- 토큰을 안전하게 보관
- 토큰이 노출되면 즉시 재생성
- 필요한 최소 권한만 부여
```

### 3. 저장소 감시
```
- 저장소에 변경사항이 있으면 이메일 알림
- Pull Request가 생성되면 알림
```

## ✅ 결론

### Public 저장소는 안전합니다!

**이유:**
1. ✅ **소유자만 직접 수정 가능**
2. ✅ **다른 사람은 Pull Request만 가능 (승인 필요)**
3. ✅ **악의적인 수정은 당신이 승인하기 전까지 반영 안 됨**
4. ✅ **2단계 인증으로 계정 보안 강화 가능**

### 웹사이트 파일의 특성
- HTML, CSS 파일만 있음
- 민감한 정보 없음
- 공개되어야 정상 (홈페이지, 개인정보처리방침 등)

---

## 🎯 권장 설정

1. ✅ **Public 저장소로 유지** (무료 GitHub Pages 사용)
2. ✅ **2단계 인증 활성화** (계정 보안)
3. ✅ **브랜치 보호 규칙 설정** (선택사항)
4. ✅ **저장소 감시 활성화** (변경사항 알림)

이렇게 설정하면 안전하게 Public 저장소를 사용할 수 있습니다! 🛡️
